# LCS-QA-FIX-20260223-02 记录：/api/status/run-once 可观测性“口径问题”

- 时间: 2026-02-23
- 优先级: P1
- 背景: QA 报告按历史默认 `127.0.0.1:8000` 采集 `/api/status/run-once`，而本服务 Web 默认端口是 `8765`，导致“不可达”的误判（口径不一致）。

## 口径结论（本修复单输出）

- **默认口径**：所有验收与可观测性采集，以 `http://127.0.0.1:8765` 为准。
- **端口口径**：产品仅支持 Web 端口 `8765`；测试/值班/回归不得使用 `8000` 作为验收口径。

## 影响范围

- 仅文档/报告口径修正，不改服务监听，不引入双端口强制绑定。

## 验收建议命令

```bash
curl -fsS http://127.0.0.1:8765/api/status/run-once | python -m json.tool
curl -fsS "http://127.0.0.1:8765/api/logs?n=20&level=INFO&module=scheduler" | python -m json.tool
```
